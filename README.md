# В процессе выполнения контрольной работы 4 были выполнены рабочие тетради 19-23

## Практическое занятие №19 - Работа с реляционными СУБД на примере PostgreSQL

Рассмотрен процесс установки, настройки и подключение реляционной СУБД на примере PostgreSQL. В приложении для управления списком пользователей, для хранения оных, была подключена реляционная СУБД PostgreSQL с помощью библиотеки Sequelize. Были определены модели и связи таблиц и созданы запросы для CRUD операций.

## Практическое занятие №20 - Работа с NoSQL СУБД на примере MongoDB

Рассмотрены различия реляционными и нереляционными СУБД на примере MongoDB и PostgreSQL. Рассмотрен процесс установки, настройки и подключение нереляционной СУБД на примере MongoDB. В приложении по управлению списком пользователей, для хранения данных, была подключена нереляционна СУБД MongoDB с помощью библиотеки Mongoose. Были определена схема, описывающая структуру модели для СУБД и созданы CRUD запросы. 

---

## Практическое занятие №21 - Кэширование с использованием Redis

Рассмотрено применения Redis в совеременных web-приложениях. На примере 11 практической тетради было рассмотрено применение Redis для временного хранения кэшированной информации о пользователях и товарах. Определены middleware для чтения данных из кэша, функции для сохранения и очистки информации в кэше.

---

## Практическое занятие №22 - Балансировка нагрузки в веб-приложениях

Рассмотрены понятие балансировки нагрузки, её виды и сценарии использования в современных web-приложениях. В приложении настроен Nginx в качестве балансировщика нагрузки: запущены три экземпляра backend-сервера на разных портах, настроен upstream с распределением запросов между серверами, добавлен резервный сервер (backup). В качестве альтернативного балансировщика настроен HAProxy с аналогичной конфигурацией.

## Практическое занятие №23 - Контейнеризация приложений с Docker

Рассмотрены ключевые концепции контейнеризации, отличие контейнеров от виртуальных машин, основные понятия Docker. Изучена структура Dockerfile и назначение его инструкций (FROM, WORKDIR, COPY, RUN, EXPOSE, CMD). На примере 22 практической тетради, backend-серверы и Nginx упакованы в Docker-контейнеры. Создан Dockerfile и написан docker-compose.yml, описывающий все сервисы. Nginx настроен на балансировку между контейнерами по их именам в сети Docker.
