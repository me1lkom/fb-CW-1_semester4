# В процессе выполнения контрольной работы 3 были выполнены рабочие тетради 13-17

## Практическое занятие №13 - Service Worker

Реализован Service Worker для кэширования статических ресурсов. Приложение полностью работает офлайн: страница загружается из кэша, заметки сохраняются в localStorage. Service Worker перехватывает сетевые запросы через событие fetch и может отдавать файлы из кэша даже при отсутствии интернета. При обновлении приложения старые кэши автоматически удаляются в событии activate.

## Практическое занятие №14 - Web App Manifest

Создан manifest.json с иконками, названием и цветовой темой. Приложение можно установить на устройство - оно запускается в отдельном окне без элементов браузера. Манифест позволяет операционной системе правильно отображать иконку приложения на рабочем столе и в меню. Для iOS добавлены отдельные meta-теги и apple-touch-icon для корректной установки на устройства Apple.

---

## Практическое занятие №15 - HTTPS + App Shell

Настроен локальный HTTPS-сервер. Реализована архитектура App Shell - каркас приложения кэшируется и отображается мгновенно, контент подгружается динамически. Без HTTPS Service Worker не работает, поэтому сгенерирован самоподписанный сертификат через mkcert. Динамические страницы (content/*) загружаются по стратегии Network First, обеспечивая актуальность контента.

---

## Практическое занятие №16 - WebSocket + Push

Разработан Node.js-сервер с Socket.io для двусторонней связи. Добавлены push-уведомления через VAPID-ключи - сервер отправляет сообщения даже при закрытом браузере. При добавлении задачи событие newTask отправляется на сервер и рассылается всем подключённым клиентам через io.emit. Push-уведомления требуют подписки пользователя и работают через Service Worker, который обрабатывает событие push.

---

## Практическое занятие №17 - Детализация Push

Добавлены отложенные напоминания с выбором даты и времени. Сервер хранит активные таймеры и отправляет push-уведомление при наступлении заданного времени. В уведомлении предусмотрена кнопка "Отложить на 5 минут". При нажатии на кнопку отправляется запрос на сервер, который отменяет старый таймер и устанавливает новый. Напоминания сохраняются в серверной памяти даже после перезагрузки страницы.
